# CLAUDE.md

このファイルはAI（Claude）との協働開発ルールを定義します。

# 【最重要】調査・質問対応の手順

**コードベースに関する質問や調査を行う際は、必ず以下の手順を守ること：**

1. **最初に `docs/` を確認する**: コードを探索する前に、まず `docs/` 配下に関連するドキュメントがないか確認する
2. **ドキュメントがあれば参照する**: 該当するドキュメントが見つかった場合は、その内容を基に回答する
3. **ドキュメントがない場合のみコード調査**: ドキュメントがない、または不十分な場合にコードベースを調査する

この手順により、最新の仕様・設計意図を正確に把握し、効率的な調査が可能になります。

# 実装計画の作成

**【絶対厳守】計画なしでコードを変更してはならない**

以下のケースも全て計画が必要：
- バグ調査のための一時的な変更（ログ追加、エラー表示追加など）
- 「小さな修正」「簡単な修正」
- 調査結果に基づく修正

**【絶対厳守】計画書の作成・修正後は必ず承認を待つ**

- 計画書を新規作成した場合 → 承認を待つ
- 計画書を修正した場合 → 再度承認を待つ
- 「はい」等の明確な承認があるまで実装に着手しない

コードの修正を行う際は、以下の手順に従ってください：

1. **既存ドキュメントを確認**: `docs/` 配下に関連する仕様・設計ドキュメントがないか確認する
2. **実装前に計画ファイルを作成**: `docs/` ディレクトリに実装計画のMarkdownファイルを作成する
  - ファイル名: <timestamp>_<number>_<実装内容>.md
  - ファイル名の例: 20260309_02_staging_basic_auth.md
3. **計画ファイルに含める内容**:
  - タスク概要
  - 現状調査結果（関連モデル、既存の処理フローなど）
  - 実装方針
  - 修正対象ファイル一覧
  - 注意事項
4. **人間に確認を求める**: 計画ファイル作成後、実装に進む前に人間の承認を得る
5. **承認後に実装開始**: 承認を得てから実装を開始する
6. **実装完了後**: 計画ファイルを `docs/plans/completed/<current-year>/<current-month>` に移動する
7. **実装内容が指示通りでない場合**: 調査を行う。原因が解明したら計画ファイルを修正し、`docs/plans/`に移動する。**人間に確認を求める**に戻る。

これにより、実装の方向性を事前に確認し、手戻りを防ぐことができます。

※ <開発プロジェクト> は該当するプロジェクト名に置き換えてください（`templates/` or `studio/`）。
