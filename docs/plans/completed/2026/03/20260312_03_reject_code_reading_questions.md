# コードリーディング問題の確実な除外

## タスク概要

プロンプトの制約追加だけではコードリーディング問題が出題され続けるため、プログラム側でも検出・リジェクトする仕組みを追加する。

## 問題の原因

LLMはプロンプトの指示を常に完全に守るとは限らない。特に小さいモデル（Haiku）では指示を無視してコードスニペットを含む問題を生成してしまう。

## 実装方針

2つの対策を組み合わせる。

### 1. プロンプト強化

現状の「生成しないでください」という表現を、より強い禁止表現に変更する。

```
【禁止】question フィールドにコードブロック（``` で囲まれた部分）を含めてはなりません。
```

### 2. 生成後のバリデーション + リトライ

生成された `question` または `choices` にコードブロック（` ``` ` または `` ` ``）が含まれている場合、コードリーディング問題とみなしてリジェクトし、最大 3 回リトライする。

```ts
function containsCode(text: string): boolean {
    return /```/.test(text);
}
```

3 回試してもコードリーディング問題が出続ける場合はエラーをスローする。

## 修正対象ファイル

- `src/quizGenerator.ts` — プロンプト強化 + バリデーション/リトライロジック追加

## 注意事項

- リトライは最大 3 回（初回含め合計 3 回）とし、無限ループを防ぐ
- バリデーションは `question` フィールドのみをチェックする（選択肢にコードが含まれるケースも念のため対象にしてよい）
